# v0.1.4 Fallback Model Wiring Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 配置中指定备用模型，主模型失败时自动接管。全局单一 fallback（决策 Q10=A）。

**Architecture:** `ModelRef` 内嵌 `Fallback *ModelRef`；`buildChatter` 静态装配，fallback 非空时 wrap 两层（`RetryWrapper`→primary + `RetryWrapper`→fallback → `FallbackWrapper`）。retry 耗尽才 fallback（Q7），provider+model 完全相等时跳过（打 warning）。

**Tech Stack:** Go 1.25+，cobra (CLI)，YAML 配置，现有 `llm.FallbackWrapper` + `llm.RetryWrapper` 结构体（已在 llm 包中）。

## Global Constraints

- Go 1.25+；module path `github.com/iannil/jianwu`（不改）。
- 每个 Task 走 TDD：先写失败测试（RED）→ 最小实现（GREEN）→ commit。
- Fallback 只在 primary retry 耗尽后才触发（Q7），4xx 不触发 retry 也不触发 fallback。
- `FallbackWrapper` 结构体已在 `internal/provider/llm/fallback.go` 中定义，`NewRetryWrapper` 在 `retry.go` 中。
- fallback provider 自身也应被 `RetryWrapper` 包裹——如果 fallback 也 transient 失败，它能 retry。
- primary == fallback（provider+model 完全相等）时打 warning + 不装 FallbackWrapper。
- 默认模式静默（slog warn），`-v` 模式打印 stderr 提示一行，`--debug` 打出完整信息。
- 配置 schema_version 保持 `"1"`（可选字段不破坏兼容，决策 Q18=C）。
- 决策依据：本文档 § 决策记录（grill-me 2026-06-26）。

## 配置变更示例

```yaml
models:
  intake:
    provider: glm
    model: glm-4.6
    # fallback: { provider: gemini, model: gemini-2.5-flash }   # 可选，注释掉
  outline:
    provider: gemini
    model: gemini-2.5-pro
    fallback:
      provider: glm
      model: glm-4.6
  scaffolding:
    provider: gemini
    model: gemini-2.5-flash
    # 无 fallback = 不装配 FallbackWrapper
  expand:
    provider: glm
    model: glm-4.6
```

## Task 1: ModelRef Fallback 字段

**Files:** `internal/config/types.go`, `internal/config/types_test.go`

**Spec:** `ModelRef` 结构体加可选 `Fallback *ModelRef` 字段。指针类型使零值（nil）表示"不配置 fallback"，与现有 YAML 解析兼容。现有测试不必改，可选字段不影响解析。

```go
type ModelRef struct {
    Provider string     `yaml:"provider"`
    Model    string     `yaml:"model"`
    Fallback *ModelRef  `yaml:"fallback,omitempty"`
}
```

## Task 2: buildChatter fallback 装配

**Files:** `internal/cli/providers.go`, `internal/cli/providers_test.go`

**Spec:** 修改 `buildChatter`：
1. 读取 `primary.Fallback`
2. 如果非 nil：
   a. 检查 primary == fallback（provider+model 全等）→ slog.Warn + 跳过装配
   b. 否则：构造 fallback provider → `NewRetryWrapper(fallbackProvider)` → `&FallbackWrapper{Primary: retryWrappedPrimary, Fallback: retryWrappedFallback}`
3. 如果 nil：原行为不变（只包 RetryWrapper）

**TDD steps:**
1. [ ] RED: 写 `TestBuildChatterWithFallback` — 配 fallback，验证返回 `*llm.FallbackWrapper`
2. [ ] RED: 写 `TestBuildChatterFallbackSelf` — primary==fallback，验证返回 `*llm.RetryWrapper`（不是 FallbackWrapper）
3. [ ] GREEN: 实现 `buildChatter` 逻辑
4. [ ] 确认现有 `TestBuildChatterIntake` / `TestBuildChatterMissingKey` 仍然通过
5. [ ] commit: `feat(cli): wire fallback in buildChatter with primary==fallback guard`
