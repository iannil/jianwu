# v0.2.0 Auto Fact-Check Implementation Plan

**Goal:** 对已展开的章节运行自动事实复核：对每个 `[^N]` 引文，读取源 URL 内容，让 LLM 验证声明是否被引文支持。输出验证结果并更新 `outline.json`。

**Architecture:** 独立命令 `jianwu factcheck <slug> <NN-MM>`，复用现有的 ProviderDeps（Reader + Chatter）。不修改 expand 管线。

**Existing Infrastructure:**
- `OutlineChapter.Claims` — LLM 自报的声明（含 `has_citation` 标志）
- `OutlineChapter.Citations` — 引文元数据（含 `URL`）
- `reader.Reader` 接口 + Jina 实现 — 已可读 URL
- `StatusExpanded` → `StatusReviewed` 状态机

## Constraints

- Go 1.25+；module path `github.com/iannil/jianwu`（不改）。
- TDD：先写失败测试（RED）→ 最小实现（GREEN）→ commit。
- 只对 `expanded` 或 `reviewed` 状态的章节可执行（需要有 Citations + Claims）。
- 不修改 expand 管线代码（只新增命令 + engine 包）。
- Reader 复用现有的 `ProviderDeps.Reader`（Jina）。
- 避免全局可变状态（沿用已完成的 DI 模式）。

## Tasks

### Task 1: FactCheck engine 包

**New files:** `internal/engine/factcheck/factcheck.go`, `types.go`, `embed.go`, `prompt/`

**Spec:** 取出一个章节的 citations 列表，逐一用 Reader 读取源内容，然后让 LLM 逐一判断每个 `claim` 是否被对应 citation 支持：

1. 输入：章节 markdown + `[]book.Citation` + `[]book.Claim` + `reader.Reader` + `llm.Chatter`
2. 遍历 citations：对每个有 URL 的 citation，调用 `reader.Read(ctx, url)` 获取文本
3. 将 claims + citation texts 打包进 prompt，让 LLM 逐一判断
4. 输出：每个 claim 的验证结果（`verified=true/false` + `reasoning`）

```go
type FactCheckInput struct {
    ChapterTitle string
    Claims       []book.Claim
    Citations    []book.Citation
}

type ClaimVerdict struct {
    ClaimText string `json:"claim_text"`
    Verified  bool   `json:"verified"`   // does the cited source support this?
    Reasoning string `json:"reasoning"`  // why or why not
    CitationID string `json:"citation_id,omitempty"`
}

type FactCheckOutput struct {
    Verdicts     []ClaimVerdict
    SourceErrors []string // URLs that failed to read
}
```

**Key design:** 可以分批处理——如果 citation 很多，每次 LLM 调用验证 3-5 个 claims + 1 个 source 内容。

### Task 3: Prompt 模板

**New files:** `internal/engine/factcheck/prompt/system.md.tmpl`, `prompt/user.md.tmpl`

**Spec:** System prompt 定义事实复核任务角色。User prompt 注入声明列表 + 引文内容。

```
你是 jianwu 的事实复核助手。
你的任务：判断每个声明是否被其引用的来源支持。

输出 JSON 数组，每个元素包含：
- claim_text: 原始声明
- verified: true/false
- reasoning: 简短解释

规则：
1. 如果声明的内容能在引文原文中找到直接或间接的证据，标记 verified=true
2. 如果引文内容与声明无关或矛盾，标记 verified=false
3. 如果不确定（引文内容不完整），标记 verified=false 并注明
```
