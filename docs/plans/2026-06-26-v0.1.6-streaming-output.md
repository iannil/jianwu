# v0.1.6 Streaming Output Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 `jianwu expand <slug> <NN-MM>` 的 draft 阶段输出流式显示 token，用户看到实时生成过程。

**Architecture:** 新增 `llm.Streamer` 接口（<-chan StreamChunk），Gemini+GLM+Mock 各自实现。`RunDraft` 加可选 `Streamer` 参数，非 nil 时走流式路径。CLI 层检测 Streamer 支持，流式输出到 stdout。

**Tech Stack:** Go 1.25+，Gemini genai SDK（`GenerateContentStream`），GLM REST + SSE 解析。

## Global Constraints

- Go 1.25+；module path `github.com/iannil/jianwu`（不改）。
- 每个 Task 走 TDD：先写失败测试（RED）→ 最小实现（GREEN）→ commit。
- 只 draft 阶段流式（Q11=D+B1）：research 和 validate 是 JSON 输出，流式无意义。
- Streamer 是可选接口 —— Chatter 不强制要求实现。不实现 Streamer 的 provider 退回到同步 Chat。
- Streaming 不做 retry（已发出的 token 无法重放）。失败时用户重跑 `expand --force`。
- `StreamChunk` 发送完毕后关闭 channel（标准 Go 模式）。
- CLI 输出到 stdout（`fmt.Print`），不换行。最终完整内容仍然写入 `.md` 文件（不变）。

## Task 1: Streamer 接口定义

**Files:** `internal/provider/llm/interface.go`

**Spec:** 在 `interface.go` 中新增：

```go
// StreamChunk is a single token from a streaming chat response.
// The channel is closed after the final chunk (Done=true or Err!=nil).
type StreamChunk struct {
	Content string // text fragment (may be empty on Done/Err)
	Done    bool   // true when stream is complete
	Err     error  // non-nil if stream terminated with error; implies Done
}

// Streamer streams chat tokens. Implementations: gemini.Provider, glm.Provider, mock.Provider.
type Streamer interface {
	Stream(ctx context.Context, req ChatRequest) (<-chan StreamChunk, error)
}
```

**Note:** 删除旧的注释占位 `// type Streamer interface { ... }`。

## Task 4: GLM Streamer 实现

**Files:** `internal/provider/llm/glm/stream.go`

**Spec:** GLM 的 OpenAI-compatible API 支持 SSE streaming。发送请求时加 `stream: true`，逐行解析 `data: {...}` SSE 事件。

需要 GLM client 加一个 `postStream` 方法返回 `io.ReadCloser`（SSE body），然后在 provider 的 `Stream` 中逐行解析：

```
data: {"choices":[{"delta":{"content":"hello"},"index":0}]}
data: {"choices":[{"delta":{"content":" world"},"index":0}]}
data: [DONE]
```

**TDD steps:**

```go
func RunDraft(
    ctx context.Context,
    chatter llm.Chatter,
    streamer llm.Streamer,  // nil = fall back to synchronous Chat
    in ExpandInput,
    dc DraftContext,
    notes ResearchNotes,

```

- `streamer != nil`：调用 `streamer.Stream(ctx, req)`，遍历 channel 拼接 `Content`，返回完整结果
- `streamer == nil`：原行为不变（`chatter.Chat`）
- `Generate` 透传 streamer（从 CLI 传入的 `ExpandInput` 或新参数）

## Task 6: CLI 流式输出

**Files:** `internal/cli/expand.go`

**Spec:** `runExpand` 中检测 `deps.Chatter` 是否也实现了 `llm.Streamer`：

```go
if streamer, ok := deps.Chatter.(llm.Streamer); ok && verbose {
    // Stream mode: print tokens to stdout as they arrive
    stream := streamer.Stream(ctx, req)
    for chunk := range stream {
        if chunk.Content != "" {
            fmt.Print(chunk.Content)
        }
        if chunk.Err != nil {
            return wrapLLMError(chunk.Err)
        }
    }
    fmt.Println() // newline after stream
} else {
    // Fallback: synchronous Chat (existing behavior)
}
```

**注意：** 
- 只在 `-v`（verbose）模式下才流式输出（用户要求看到 token 时才看）
- 无 `-v` 时走原有 `expand.Generate` 路径，静默产出文件
- 流式模式也需要把完整结果写入 `.md` 文件（原有逻辑不变）
