## 切片目标

把 v0.3.5 审计发现的 4 项实施层缺口集中处理，让"声称 v0.3.5"变成"可验证 v0.3.5"：

1. 用户能 `jianwu --version` 查询版本号（解决 F1）
2. 真实 release 流程能产出带 version 的二进制 + git tag（解决 F1）
3. 用户能查全书累计 token，outline/scaffolding 也有 `--tokens`（解决 F2）
4. v0.3.1 进度模型 + v0.3.2 TokenTracker 有测试覆盖（解决 F3 + F5）

## 不在本切片范围

- 补打 v0.1.4–v0.3.5 缺失 tag（F1 决定不做，历史就过去）
- GitHub Actions 自动化（YAGNI，等 mouqin 真上线再评估）
- 成本估算（Q4 决定不做）
- v0.4 多租户接线（推迟到触发式启动）

- **T8 release.sh 测试**：本地 dry-run + tag 模拟，不真推 remote（避免污染 tag）。
