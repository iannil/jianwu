## Task 1 — 提取 `findNextDimension` 消除重复 Walk（P1）

**现状:** `grill.go` 中 `Run` 函数在行 31-44 和行 86-93 有两段几乎相同的 walk+find-pending 逻辑。

**方案:**
1. 在 `tree.go` 中新增方法 `func (t *DesignTree) NextPending(answers map[string]string) *Dimension`
2. 实现：调用 `t.Walk(answers)`，返回第一个未回答的维度（不在 answers 中），若全部答完返回 nil
3. `grill.go` 两个位置调用 `tree.NextPending(session.Answers)` 替代重复逻辑

**测试:**
- `tree_test.go`: 新增 `TestNextPending`：空 answers 返回 topic，部分 answers 返回下一个未答，全答完返回 nil
- `grill_test.go`: 现有测试继续通过（行为不变）

```go
// tree.go
func (t *DesignTree) NextPending(answers map[string]string) *Dimension {
	for i := range t.Dimensions {
		if _, answered := answers[t.Dimensions[i].ID]; !answered {
			// Must check if this dim is in the walk order given current answers.
			// We need walk to respect deps + triggers.
			walk := t.Walk(answers)
			for _, wd := range walk {
				if _, ok := answers[wd.ID]; !ok {
					return &wd // first un-answered in walk order
				}
			}
		}
	}
	return nil
}
```

## Task 2 — 推荐值校验 Options 合法性（P2）

**现状:** `Run` 接受用户的任意字符串，不做 Options 校验。若 LLM 输出格式不匹配，非法值会传递到 outline/scaffolding。

**方案:**
1. 在 `tree.go` 中新增 `func (d *Dimension) ValidateAnswer(answer string) bool`
2. 如果 `d.Options` 为空（如 topic），任何值都合法；否则检查是否在 Options 中
3. 在 `Run` 的 `final` 确定后、`session.RecordAnswer` 前插入校验
4. 校验失败时：如果 `d.DefaultValue != ""` 则回退到默认值，否则保留原值（不阻断流程）

**注意：** topic 的 Options 为 `[]string{}`（空），所以任何值都通过校验。audience 等有 Options 列表的维度才会触发校验。

**测试:**
- `tree_test.go`: 新增 `TestValidateAnswer`：合法值通过、非法值拒绝、空 Options 接受任意值
- `grill_test.go`: 新增 `TestRunValidatesAnswerAgainstOptions`：mock 返回非法值，验证回退或保留

```go
// tree.go
func (d *Dimension) ValidateAnswer(answer string) bool {
	if len(d.Options) == 0 {
		return true // free text
	}
	for _, opt := range d.Options {
		if opt == answer {
			return true
		}
	}
	return false
}
```

## Task 3 — Context 取消检查（P5）

**现状:** `Run` 不检查 `ctx.Err()`，在 LLM 推荐完成后若用户已取消，仍会执行业务逻辑。

**方案:**
1. 在 `Run` 中两处关键阶段插入 context 检查：
   - `Walk` 之后、`Recommend` 之前：`select { case <-ctx.Done(): return nil, ctx.Err() }`
   - `Recommend` 之后、`ui.Ask` 之前：同上

**测试:**
- `grill_test.go`: 新增 `TestRunRespectsContextCancellation`：使用已取消的 context，验证 Run 立即返回 ctx.Err()

```go
select {
case <-ctx.Done():
	return nil, ctx.Err()
default:
}
```

## Task 4 — 空推荐值解析边界处理（P6）

**现状:** `grill.go:60-74` 接受空字符串时取推荐第一行；若推荐无换行符则 fallback 取全文本。行为隐式。

**方案:**
确保解析逻辑更明确：
- 提取为辅助函数 `func extractFirstLine(s string) string`
- 在注释中明确文档化解析规则
- 当前行为不变（取第一行→无换行则取全文），仅做提取+注释

**测试:**
- `grill_test.go`: 新增 `TestRunExtractFirstLine` 边界测试

```go
// extractFirstLine returns the first line of s.
// Used to parse the recommended value from LLM output (first line = value, rest = reasoning).
func extractFirstLine(s string) string {
	for i, r := range s {
		if r == '\n' {
			return s[:i]
		}
	}
	return s
}
```
